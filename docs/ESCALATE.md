# ESCALATE — 人机协作（Human-in-the-Loop）升级路径

## 触发条件

| 条件 | 升级级别 | 操作 |
|-----------|-----------------|--------|
| 请求特权工具调用 | 级别 1 | 等待人工批准（超时时间：5 分钟） |
| 代码部署到生产环境 | 级别 2 | 需要双重批准 |
| 访问新域名的网络权限 | 级别 2 | 需要更新域名白名单 + 批准 |
| 在项目目录外写入文件 | 级别 1 | 等待人工批准 |
| L3+ 级别自我进化补丁 | 级别 3 | 需要单人批准 + 完整回归测试 + 影子对比报告（shadow diff report） + 强制冷却期 |
| 超过预算阈值 | 级别 1 | 通知 + 等待确认 |

## 审批流程

1. 系统向黑板（blackboard）发出 HITL 检查点事件
2. 通过配置的渠道发送通知（CLI 提示 / HTTP 回调）
3. 人工在超时窗口内批准/拒绝
4. 超时处理：拒绝（安全默认设置）
5. 审批记录在不可变的审计追踪（audit trail）中

## 绕过（仅限紧急情况）

拥有管理员密钥的物理控制台访问可以覆盖审批。所有的绕过操作都会记录原因和身份信息。
