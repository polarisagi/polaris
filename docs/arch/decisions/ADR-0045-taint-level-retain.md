# ADR-0045: 保留五级污点传播（GD-13-004 否决 / GD-14-003 采纳）

- **状态**: Accepted（裁决完成，无代码改动，维持现状）
- **日期**: 2026-07-11
- **决策者**: MrLaoLiAI
- **相关模块**: M11 / `internal/security/taint`

## 上下文

Gemini 设计评审同一批报告内给出两条相互矛盾的结论：

- **GD-13-004**：建议把当前五级污点等级（`TaintNone/TaintLow/TaintMedium/TaintHigh/TaintCritical`，具体命名以 `pkg/types` 实际定义为准）合并简化为三态，理由是"五级粒度在实践中大多数路径只用到二值判断，增加了不必要的复杂度"。
- **GD-14-003**：建议保留现有五级污点传播，理由是"细粒度污点等级是可验证安全边界的基础（对应业界 NIST AI RMF / NeMo Guardrails 等参考框架的分级实践），合并会造成安全粒度倒退"。

## 决策

**否决 GD-13-004，采纳 GD-14-003**：维持现有五级污点传播机制不变，本 ADR 不产生任何代码改动。

裁决依据：

1. 两条建议直接矛盾，需要择一。
2. GD-14-003 给出的置信度更高（报告原文标注"高" vs GD-13-004 的"低"），且引用了更具体的业界对比依据（NIST AI RMF、NeMo Guardrails 的分级实践）。
3. GD-13-004 的论据是"大多数路径只用到二值判断"这一理论推演，未附带量化证据（例如埋点统计各污点等级在真实流量中的实际占比分布）。
4. 污点传播是 HE-2（可验证执行）/ HE-7（防退化边界）两条核心不变量的直接实现基础（`internal/security/taint`）。在没有量化证据支撑"五级在生产环境中确实退化为事实上的二值"之前，合并简化属于安全粒度的主动倒退，代价高、收益不确定。

## 后果

- 正向：安全边界粒度不倒退，`internal/security/taint` 五级传播逻辑及其下游消费方（Cedar 策略、PromptBuilder 注入前校验等）保持现状，零改动风险。
- 负向：若 GD-13-004 描述的"多数路径只用二值判断"确实存在，当前设计在这些路径上仍背负五级判断的复杂度成本；但这是可接受的存量设计权衡，不构成否决本次裁决的理由。

## 重新评估触发条件

未来若有人重提"简化污点等级"的建议，必须先提供以下量化证据，而不是理论推演，才应重新评估：

- 埋点统计 `TaintMedium`/`TaintLow` 等中间等级在真实生产流量中的实际占比、以及这些等级触发的差异化处理路径（如果占比极低且下游从未有过差异化处理分支被实际命中，才具备简化的实证基础）。
- 因五级粒度本身（而非其他原因）导致的具体维护成本案例（如反复出现的等级判断 bug、开发者常见误用模式的统计）。

满足以上证据后，方可重新提交设计评审，且需同时评估安全侧影响（不能只看开发体验收益）。
